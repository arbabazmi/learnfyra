# M08 — LLM Generation & Routing System
**Module:** M08
**Status:** Pending Implementation
**Version:** 1.0
**Last Updated:** 2026-03-29

---

## Overview

M08 provides a production-grade question generation pipeline with smart model routing,
answer validation, response caching, and batch generation. It powers both the question bank
(M02) and worksheet generation (M03) by selecting the most cost-effective model per
request type and validating all generated answers before storage.

## Routing Rules

### Model IDs (Anthropic SDK)
- Fast / cheap: `claude-haiku-4-5-20251001`
- Smart / accurate: `claude-sonnet-4-6`

### Route to Haiku
- question type: `multiple-choice`, `true-false`, `fill-in-the-blank`
- difficulty: Easy or Medium (grades 1–8, Math / ELA)
- step: explanation generation

### Route to Sonnet
- question type: `word-problem`, `show-your-work`, `short-answer`
- difficulty: Hard (any grade/subject)
- step: answer validation (always)
- grade 9–10 (any type, any difficulty)
- fallback when Haiku result fails validation twice

---

## Generation Pipeline (per question)

Step 1 — Generate question JSON (model selected by router)
Step 2 — Generate answer JSON (model selected by router)
Step 3 — Validate answer (always Sonnet; returns is_correct + confidence + corrected_answer)
Step 4 — Generate explanation JSON (always Haiku)

If Step 3 fails (is_correct=false or confidence < 0.7):
- Retry Step 1+2 with same model (attempt 1)
- If still fails, escalate to Sonnet for Step 1+2 (attempt 2)
- If still fails after 2 retries, log warning and use best available answer

## Self-Consistency (optional, default off)

When `selfConsistency: true` is passed:
- Generate 2 independent answers
- If they match, use that answer
- If they differ, run validation on both and pick higher-confidence result

---

## Cache Design

Cache key: `grade:subject:questionType:difficulty`

- In-memory Map (TTL 1 hour default)
- On GET: check if at least the requested count of questions exist in cache
- On MISS: generate new questions, add to cache pool
- On HIT: return from pool (splice to avoid reuse within same session)

---

## Batch Generation

- `generateBatch({ grade, subject, questionType, difficulty, count })` returns array of question objects
- Concurrency: max 3 simultaneous pipeline calls (p-limit style)
- Deduplication: skip questions whose dedupeHash already exists in cache

## Functional Requirements

### REQ-M08-001: Model Router (src/ai/routing/modelRouter.js)
**Priority:** P0
**Tasks:** M08-BE-01

`src/ai/routing/modelRouter.js` SHALL export a single function `selectModel(grade, subject, questionType, difficulty, step)` that returns either `"haiku"` or `"sonnet"` based on the following precedence rules:

1. If `step === "validation"`, always return `"sonnet"`.
2. If `grade >= 9`, always return `"sonnet"`.
3. If `questionType` is one of `["word-problem", "show-your-work", "short-answer"]`, return `"sonnet"`.
4. If `difficulty === "Hard"`, return `"sonnet"`.
5. Otherwise (MCQ / true-false / fill-in-the-blank, Easy or Medium, grades 1–8), return `"haiku"`.
6. Explanation step always uses `"haiku"` regardless of other parameters.

### REQ-M08-002: Answer Validator (src/ai/validation/answerValidator.js)
**Priority:** P0
**Tasks:** M08-BE-02

`src/ai/validation/answerValidator.js` SHALL export `validateAnswer(question, answer, context)` that:
1. Always calls Sonnet (ignores router for this step).
2. Sends the question object, proposed answer, and optional context to the validation prompt.
3. Returns a `ValidationResult` object: `{ is_correct, confidence, corrected_answer }`.
4. If the API call fails, wraps the error and re-throws — never silently swallows validation failures.

### REQ-M08-003: Question Cache (src/ai/cache/questionCache.js)
**Priority:** P1
**Tasks:** M08-BE-03

`src/ai/cache/questionCache.js` SHALL export a `QuestionCache` class with the following interface:
- `get(grade, subject, questionType, difficulty, count)` — returns an array of up to `count` questions or `null` on cache miss.
- `set(grade, subject, questionType, difficulty, questions)` — adds questions to the pool for the given key.
- `has(grade, subject, questionType, difficulty, count)` — returns `true` if at least `count` questions are available in the pool.
- `flush()` — clears the entire cache (used in tests).
- TTL defaults to 1 hour. Questions are spliced from the pool on GET to prevent reuse within the same session.

### REQ-M08-004: Question Pipeline (src/ai/pipeline/questionPipeline.js)
**Priority:** P0
**Tasks:** M08-BE-07

`src/ai/pipeline/questionPipeline.js` SHALL export `runPipeline(params)` that executes the 4-step generation pipeline per question. Retry/escalation logic SHALL be:
- Attempt 1: use model chosen by router.
- If validation fails (is_correct=false or confidence < 0.7): retry Step 1+2 with the same model.
- Attempt 2: if still failing, escalate to Sonnet.
- After 2 retries, log a warning and use the best available answer — never throw an error that stops the batch.

### REQ-M08-005: Batch Generator (src/ai/pipeline/batchGenerator.js)
**Priority:** P0
**Tasks:** M08-BE-08

`src/ai/pipeline/batchGenerator.js` SHALL export `generateBatch(params)` that:
1. Accepts `{ grade, subject, questionType, difficulty, count }`.
2. Checks the cache first via `QuestionCache.has()` — if enough questions cached, return from cache.
3. For each question to generate, runs `runPipeline` with a concurrency limit of 3 simultaneous calls.
4. Skips questions whose `dedupeHash` already exists in the cache pool.
5. Returns an array of `GeneratedQuestion` objects.

### REQ-M08-006: Express Route and Lambda Handler
**Priority:** P0
**Tasks:** M08-BE-09, M08-BE-10

`backend/handlers/generateQuestionsHandler.js` SHALL be a Lambda-compatible handler for `POST /api/generate-questions`. It SHALL:
1. Validate the request body: `grade` (1–10 integer), `subject` (enum), `topic` (non-empty string), `difficulty` (enum), `questionCount` (5–30 integer), `questionType` (enum). Return 400 with `INVALID_REQUEST` on any validation failure.
2. Call `generateBatch` with the validated parameters.
3. Return 200 with the full response including `questions`, `metadata`, and `costTracking`.
4. Handle OPTIONS preflight per the standard CORS pattern.
5. Return 500 with `{ error: message }` on unexpected failures.

`server.js` SHALL be updated to add a `POST /api/generate-questions` route that wraps the handler in the same adapter pattern used by the existing `/api/generate` route.

Lambda configuration (future AWS): `learnfyra-generate-questions`, timeout 30 s, memory 512 MB, ARM_64.

## Acceptance Criteria

### AC-M08-001: Haiku Route — MCQ Easy/Medium Grades 1–8
**Given** grade=3, subject=Math, type=multiple-choice, difficulty=Easy
**When** the router is called
**Then** the returned model is `"haiku"`

### AC-M08-002: Sonnet Route — word-problem or Hard
**Given** type=word-problem OR difficulty=Hard
**When** the router is called
**Then** the returned model is `"sonnet"`

### AC-M08-003: Sonnet Route — Validation Step Always
**Given** step=validation
**When** the router is called regardless of question type
**Then** the returned model is `"sonnet"`

### AC-M08-004: Retry on Validation Failure
**Given** a generated answer where is_correct=false
**When** the pipeline processes it
**Then** it retries up to 2 times before falling back to best available answer

### AC-M08-005: Cache Hit Skips LLM Calls
**Given** 10 questions requested
**When** all 10 exist in cache
**Then** no LLM calls are made and the response cacheHits value equals 10

### AC-M08-006: POST /api/generate-questions Returns Full Response
**Given** POST /api/generate-questions with valid body
**When** the request completes
**Then** the response includes a `questions` array and a `costTracking` object

### AC-M08-007: Sonnet Route — Grades 9–10
**Given** grade=9 or grade=10 with any type and any difficulty
**When** the router is called
**Then** the returned model is `"sonnet"`

### AC-M08-008: Invalid Request Returns 400
**Given** POST /api/generate-questions with grade=0 or questionCount=100
**When** the handler processes the request
**Then** the response is 400 with code `INVALID_REQUEST`

### AC-M08-009: Concurrency Limit Respected
**Given** a batch of 10 questions is requested
**When** the batch generator runs
**Then** no more than 3 pipeline calls are active simultaneously

### AC-M08-010: CORS Preflight Returns 200
**Given** an OPTIONS request to POST /api/generate-questions
**When** the handler processes the request
**Then** the response is 200 with CORS headers present

## Local Development Notes

- Set `ANTHROPIC_API_KEY` in `.env`.
- Cache is in-memory per process and resets on server restart.
- No DynamoDB required for M08 (question bank storage is M02's concern).
- Questions generated by the M08 pipeline can be fed into M02's bank.
- `NODE_ENV=development` — no special runtime switch needed beyond the API key.
