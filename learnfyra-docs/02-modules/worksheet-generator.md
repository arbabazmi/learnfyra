# M02 + M03 — Question Bank & Worksheet Generator

## Overview

M02 is the Question Bank — a DynamoDB-backed store of reusable, deduplicated questions indexed by grade, subject, topic, type, and difficulty.

M03 is the Worksheet Generator — it implements bank-first assembly: query M02 first, then call Claude AI for gap-fill only when the bank has insufficient questions.

These two modules are tightly coupled and documented together because M02 exists to serve M03.

---

## M02 — Question Bank

### Purpose

- Store every question ever generated by Claude AI for reuse
- Prevent duplicate questions via SHA256 dedupeHash
- Enable per-student repeat caps (Phase 2)
- Reduce Claude API cost on subsequent generations of the same topic

### DynamoDB Table Schema

```
Table: LearnfyraQuestionBank-{env}
Billing: PAY_PER_REQUEST
PK: questionId (String, UUID v4)
```

| Attribute | Type | Description |
|---|---|---|
| questionId | String | UUID v4, PK |
| dedupeHash | String | SHA256(grade\|subject\|topic\|type\|normalizedQuestionText) |
| grade | Number | 1–10 |
| subject | String | Math \| ELA \| Science \| Social Studies \| Health |
| topic | String | e.g., "Multiplication" |
| type | String | multiple-choice \| fill-in-the-blank \| etc. |
| difficulty | String | Easy \| Medium \| Hard |
| question | String | Question text |
| options | List | A/B/C/D options (multiple-choice only) |
| answer | String | Correct answer |
| explanation | String | Explanation for answer key |
| points | Number | 1 (default) |
| standards | List | CCSS / NGSS / NHES codes |
| createdAt | String | ISO-8601 timestamp |
| generatedByModel | String | Claude model that generated this question |
| timesUsed | Number | Usage counter (for analytics) |

### GSI-1 (for question lookup during generation)

```
Index name: GSI-1
PK: lookupKey = "grade#{grade}#subject#{subject}#topic#{topic}"
SK: typeDifficulty = "{type}#{difficulty}"
```

Example: `lookupKey = "grade#3#subject#Math#topic#Multiplication"`, `typeDifficulty = "multiple-choice#Medium"`

### dedupeHash Calculation

```javascript
import { createHash } from 'crypto';

function computeDedupeHash(question) {
  const normalized = [
    question.grade,
    question.subject.toLowerCase(),
    question.topic.toLowerCase(),
    question.type,
    question.question.toLowerCase().replace(/\s+/g, ' ').trim()
  ].join('|');

  return createHash('sha256').update(normalized).digest('hex');
}
```

## M03 — Worksheet Generator

### Generation Pipeline

```
POST /api/generate
  {grade, subject, topic, difficulty, questionCount, formats, studentId?}

Step 1: Validate input
  grade: 1-10
  subject: Math | ELA | Science | Social Studies | Health
  difficulty: Easy | Medium | Hard | Mixed
  questionCount: 5-30
  formats: subset of [pdf, docx, html]

Step 2: Determine required question breakdown
  For each question type relevant to this grade/subject:
    targetCount[type] = Math.ceil(questionCount * typeRatio[type])

Step 3: Query Question Bank (per type)
  For each type:
    banked = QB.getQuestions({grade, subject, topic, type, difficulty, count: targetCount[type]})

Step 4: Determine generationMode
  totalBanked = sum of all banked
  if totalBanked >= questionCount: generationMode = 'bank-only'
  if totalBanked = 0:              generationMode = 'ai-only'
  else:                            generationMode = 'mixed'

Step 5: Call Claude AI (if generationMode != 'bank-only')
  gap = questionCount - totalBanked
  buildSystemPrompt() + buildUserPrompt({...options, questionCount: gap})
  model: read from Config table (default: claude-sonnet-4-20250514)
  max_tokens: 8192
  Retry up to 3 times with strict prompt if validation fails

Step 6: Validate and coerce AI response
  extractJSON(rawText) → JSON.parse → coerceTypes → validateTopLevel → validateQuestions

Step 7: Save new AI questions to bank
  For each new question:
    dedupeHash = computeDedupeHash(question)
    if !QB.questionExists(dedupeHash): QB.saveQuestion(question)

Step 8: Assemble final worksheet JSON
  Merge banked + AI questions, shuffle if difficulty = 'Mixed'
  Add: worksheetId (UUID v4), generatedAt (ISO-8601), timerSeconds
  provenanceLevel: full-bank | partial-bank | full-ai

Step 9: Export to requested formats (parallel)
  htmlExporter.js → worksheet.html
  pdfExporter.js  → worksheet.pdf
  docxExporter.js → worksheet.docx
  answerKey.js    → answer-key.html/pdf/docx

Step 10: Write to storage
  APP_RUNTIME=aws:   S3 PUT to worksheets/{year}/{month}/{day}/{uuid}/
  APP_RUNTIME=local: write to worksheets-local/{uuid}/

Step 11: Return
  {worksheetId, downloadUrls, solveUrl, estimatedTime,
   generationMode, provenanceLevel, questionCount}
```

### Question Types by Grade Band

| Question Type | Grades | Notes |
|---|---|---|
| multiple-choice | 1-10 | Exactly 4 options (A/B/C/D) |
| true-false | 1-8 | Options: ["True", "False"] |
| fill-in-the-blank | 1-10 | Short text answer |
| short-answer | 3-10 | 1-3 sentence answer |
| matching | 2-10 | 4-6 pairs |
| show-your-work | 4-10 | Work area + final answer |
| word-problem | 2-10 | Scenario + numeric or written answer |

### Curriculum Map (src/ai/topics.js)

Topics are verified against official curriculum standards. The CURRICULUM map contains 410 topic combinations, minimum 8 topics per grade/subject entry.

Coverage:
- Math/ELA: CCSS (Common Core State Standards)
- Science: NGSS (Next Generation Science Standards)
- Social Studies: C3 Framework
- Health: NHES (National Health Education Standards)

All Claude prompts include the relevant standard codes for the requested topic. No content is generated outside the verified curriculum map.

### Always-Available Offline Downloads

PDF and Word (DOCX) downloads are always available for every generated worksheet. Users can download worksheets in these formats for offline use at any time, regardless of whether they use online solving features. This ensures access to printable and editable versions for all users.
